Exercício #1

Objetivos:

  • Praticar Conceitos de Engenharia de Software
  • Simular situações enfrentadas na fase de Concepção de um Projeto (Inception)
  • Executar Atividades e Produzir Artefatos referentes a esta mesma fase

O Exercício Proposto:

  1. Organizem-se em grupos, mantendo a mesma equipe de Projeto A durante todo o transcorrer da disciplina neste semestre.  
    • Se seu grupo não faz Projeto A, o trabalho deve usar o estudo de caso dado. Utilize os quatro artefatos já produzidos e configurados na versão 1.0 e disponíveis na página do estudo de caso.
  2. Neste exercício, a equipe deverá desempenhar todas as atividades definidas nos workflows da fase de Concepção (Requisitos, Análise e Projeto e Gerenciamento)  que forem necessárias para elaborar os artefatos abaixo listados: 

o       Capa do Trabalho

o       Visão Geral do Sistema (este template já foi preenchido pelo professor na proposta do estudo de caso para os que não cursam Projeto A)

o       Descrição Introdutória dos Requisitos (este template já foi preenchido pelo professor na proposta do estudo de caso para os que não cursam Projeto A)

o       Lista de Atores e Casos de Uso (este template já foi preenchido pelo professor na proposta do estudo de caso para os que não cursam Projeto A)

o       Modelo de Casos de Uso (este template já foi preenchido pelo professor na proposta do estudo de caso para os que não cursam Projeto A)

o       Esboço de Caso de uso  (todos os casos de uso do Projeto)

o       Esboço de Arquiteturas Candidatas

o       Lista de Riscos

o       Relatório de Análise de Risco

o       Plano de Fases e Iterações

o       Estimativas de Projeto

 

  1. Apresente um Relatório em fomato WORD contendo todos os artefatos produzidos através dos templates acima (os quatro que foram dados pelo professor e todos os sete produzidos pelo grupo), na data definida no calendário da disciplina.
